Tutorial: Modelado Predictivo

Una inmersión profunda en la anatomía de los algoritmos. Descubre línea por línea cómo construimos Regresiones, Árboles y Bosques Aleatorios para resolver problemas reales.

1. Regresión Múltiple (Statsmodels)

Practica2.py

El Problema: Queremos predecir un valor numérico continuo (ej. cuántas toneladas de cultivo se van a cosechar) basándonos en factores ambientales como el riego, la temperatura y la humedad.

1

La Fórmula Matemática (Patsy)

En Statsmodels, usamos una sintaxis especial de strings: "Rendimiento_t_ha ~ Riego_mm + Temperatura_C". La virgulilla (~) significa "depende de". Le estamos diciendo al modelo que el rendimiento de la cosecha depende de el riego y la temperatura.

2

El Entrenamiento (.fit)

El comando .fit() aplica el método de Mínimos Cuadrados Ordinarios (OLS). Matemáticamente, traza millones de líneas imaginarias hasta encontrar la que pasa más cerca de todos los puntos de datos a la vez, minimizando el margen de error.

3

El Reporte (Summary)

A diferencia de otras librerías, .summary() es tu herramienta de auditoría. Si en el reporte ves que la variable "Humedad" tiene un P>|t| mayor a 0.05, significa que estadísticamente la humedad fue pura coincidencia y no afecta realmente tu cosecha. ¡Esa es tu respuesta de examen!

Predicción vs Realidad 


import pandas as pd 
import statsmodels.formula.api as smf 

df = pd.read_csv("Documentos/agricultura.csv")

# PASO 1 y 2: Definir la ecuación y entrenar el modelo
modelo = smf.ols(
    formula="Rendimiento_t_ha ~ Riego_mm + Temperatura_C + Humedad", 
    data=df
).fit()

# PASO 3: Imprimir el reporte estadístico gigante
print(modelo.summary())

# Predicción: Inyectar las respuestas matemáticas en una nueva columna
df["Estimacion"] = modelo.predict(df)

2. Árboles de Clasificación (Sklearn)

Practica5.py

El Problema: Ahora la respuesta no es un número. Queremos saber si el agua de un pozo es "Apta" o "No Apta" (Clasificación Categórica). Para esto, el modelo hace una serie de preguntas de Sí/No como en el juego "Adivina Quién".

1

LabelEncoder (El Traductor)

Las matemáticas no entienden la palabra "Pozo" o "Río". LabelEncoder.fit_transform() lee esas palabras y las convierte en números (Ej: Pozo = 0, Río = 1). Regla de oro: Todo texto debe ser codificado antes de entrar al modelo.

2

Train/Test Split (La Prueba Ciega)

Si le damos al modelo todas las respuestas, solo las memorizará. Con test_size=0.3, ocultamos el 30% del dataset en una bóveda. El modelo estudia con el 70% restante (Entrenamiento) y luego le tomamos un examen sorpresa con el 30% oculto (Prueba).

3

Controlando el Overfitting

¿Qué pasa si el árbol hace demasiadas preguntas? Se vuelve súper específico y falla con datos nuevos (Sobreajuste). El parámetro max_depth=3 le corta las ramas, obligándolo a hacer solo 3 niveles de preguntas, forzándolo a encontrar patrones generales, no excepciones. 


from sklearn.model_selection import train_test_split 
from sklearn.preprocessing import LabelEncoder 
from sklearn.tree import DecisionTreeClassifier 
from sklearn.metrics import classification_report

df = pd.read_csv("Documentos/CalidadAgua.csv")

# PASO 1: Traducir texto a números
le_Apta = LabelEncoder()
df["Apta"] = le_Apta.fit_transform(df["Apta"])

# Separar variables (X) de la respuesta que queremos adivinar (y)
X = df.drop("Apta", axis=1)
y = df["Apta"]

# PASO 2: División 70/30 para evitar trampa
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=40)

# PASO 3: Entrenar con poda (max_depth)
modelo = DecisionTreeClassifier(max_depth=3)
modelo.fit(X_train, y_train)

# Evaluación Final
y_pred = modelo.predict(X_test)
print(classification_report(y_test, y_pred))

3. Random Forest (Ensamble Avanzado)

PracticaDiabetes.py

La Evolución: Cuando trabajamos con diagnósticos médicos (como predecir Diabetes), un solo Árbol de Decisión es muy riesgoso; si se equivoca en la primera pregunta, todo el diagnóstico está mal. La solución es crear un Bosque Aleatorio.

1

n_estimators (La Sabiduría de las Multitudes)

En lugar de crear 1 árbol, instanciamos RandomForestClassifier(n_estimators=10000). Esto crea 10,000 árboles distintos, cada uno entrenado con partes ligeramente diferentes de los datos. Para dar un diagnóstico, los 10,000 árboles votan. La clase con más votos gana. Esto aniquila el margen de error.

2

Feature Importances

Una vez que el bosque está entrenado, le podemos preguntar: "De todas las variables médicas que te di, ¿cuál influyó más en tus decisiones?". El comando modelo.feature_importances_ nos devuelve un porcentaje exacto. Descubriremos que la Glucosa y el BMI pesan mucho más que la Presión Arterial. 


# Importamos el algoritmo de Ensamble (Bosque)
from sklearn.ensemble import RandomForestClassifier

# PASO 1: Plantar 10,000 árboles que voten juntos
modelo = RandomForestClassifier(
    n_estimators=10000, 
    random_state=40
)
# El bosque aprende de forma robusta
modelo.fit(X_train, y_train)

# Evaluamos su precisión
y_pred = modelo.predict(X_test)
print(classification_report(y_test, y_pred))

# PASO 2: Extraer la importancia de cada síntoma
importancias = modelo.feature_importances_
for columna, imp in zip(X.columns, importancias):
    print(f"{columna}: {imp * 100:.2f}%")
